## Visual Studio 사용자를 위한 C++ 프로젝트 빌드 및 실행 가이드

> 이 프로젝트는 원래 macOS 환경에서 만들어졌기 때문에, 
> Windows의 Visual Studio에서 실행하려면 몇 가지 설정 단계가 필요합니다. 
> 아래 순서대로 따라 하면 쉽게 빌드하고 실행할 수 있습니다.

### 사전 준비
```
Visual Studio 설치 확인: Visual Studio에 "C++를 사용한 데스크톱 개발" 워크로드가 설치되어 있는지 확인해주세요. 설치되어 있지 않다면 Visual Studio Installer를 통해 추가해야 합니다.

🏠 압축 해제: 전달받은 압축 파일을 원하는 곳에 풀어주세요. 
src 폴더(소스 코드)와 include 폴더(헤더 파일)가 가장 중요합니다.

🏠 1단계: Visual Studio에서 새 프로젝트 생성
- Visual Studio를 실행하고 **"새 프로젝트 만들기"**를 선택합니다.
- 프로젝트 템플릿에서 **"콘솔 앱(Console App)"**을 선택합니다. C++용 템플릿인지 꼭 확인하세요.
- 프로젝트 이름(예: cppEx)과 위치를 정하고 **"만들기"**를 클릭합니다.

🏠 2단계: 기존 소스 파일 및 헤더 파일 추가
- 새 프로젝트가 만들어지면, 기본으로 생성된 cppEx.cpp 같은 파일은 필요 없으니 삭제해도 좋습니다. 
- 이제 우리가 받은 파일들을 프로젝트에 추가할 차례입니다.

🏠 파일 복사: 압축 해제한 폴더에서 src와 include 폴더를 복사하여, 
방금 Visual Studio가 생성한 프로젝트 폴더 안에 붙여넣습니다. 
(솔루션 탐색기에서 프로젝트를 우클릭하고 "파일 탐색기에서 폴더 열기"를 선택하면 쉽게 찾을 수 있습니다.)

🏠 소스 파일 추가:

Visual Studio의 솔루션 탐색기에서 "소스 파일" 필터를 마우스 오른쪽 버튼으로 클릭합니다.
**추가 > 기존 항목...**을 선택합니다.
방금 복사한 src 폴더로 이동하여 
- main.cpp, 
- vectorEx.cpp, 
- helloEx.c 등 c, cpp 파일을 모두 선택하고 "추가" 버튼을 누릅니다.


🏠 헤더 파일 추가:

- 솔루션 탐색기에서 "헤더 파일" 필터를 마우스 오른쪽 버튼으로 클릭합니다.
- **추가 > 기존 항목...**을 선택합니다.
- include 폴더로 이동하여 helloEx.h, vectorEx.h 파일을 모두 선택하고 "추가" 버튼을 누릅니다.


🏠 3단계: 프로젝트 설정 변경
컴파일러가 헤더 파일을 찾고, 올바른 C++ 표준을 사용하도록 두 가지 중요한 설정을 해야 합니다.

솔루션 탐색기에서 프로젝트 이름(cppEx)을 마우스 오른쪽 버튼으로 클릭하고 맨 아래의 **"속성"**을 선택합니다.

🏠 헤더 파일 경로 설정:

- 속성 페이지 창에서 C/C++ > 일반으로 이동합니다.
- "추가 포함 디렉터리" 항목을 선택하고 오른쪽의 드롭다운 화살표를 눌러 <편집...>을 엽니다.
- 새 줄 아이콘을 클릭하고 $(ProjectDir)include 라고 입력합니다. 
- 이는 프로젝트 폴더 내의 include 폴더를 헤더 검색 경로에 추가하라는 의미입니다.
- **"확인"**을 눌러 저장합니다.

🏠 C++ 표준 설정:

- 속성 페이지 창에서 C/C++ > 언어로 이동합니다.
- "C++ 언어 표준" 항목을 ISO C++17 표준 (/std:c++17)으로 변경합니다.
- "적용" 후 **"확인"**을 눌러 속성 창을 닫습니다.
참고: 우리 프로젝트는 C++ 코드(.cpp)와 C 코드(.c)를 함께 사용합니다. 
     helloEx.h 파일에 있는 extern "C" 구문 덕분에 
     Visual Studio의 C++ 컴파일러가 이를 자동으로 올바르게 처리해주므로 추가 설정은 필요 없습니다.

🏠 4단계: 빌드 및 실행
- 모든 설정이 끝났습니다! 이제 프로젝트를 빌드하고 실행할 수 있습니다.

🏠 빌드 및 디버깅 시작: 
- 키보드에서 F5 키를 누릅니다. 
  Visual Studio가 코드를 컴파일(빌드)하고 디버거와 함께 프로그램을 실행합니다.
- 디버깅 없이 실행: 키보드에서 Ctrl + F5를 누릅니다. 
  빌드 후 디버거 없이 프로그램이 바로 실행되며, 
  실행이 끝나도 콘솔 창이 바로 닫히지 않아 결과를 확인하기 편합니다.
- 이제 터미널 창이 나타나고 익숙한 메뉴가 보일 것입니다!

🏠 전달 파일 요약
친구에게 프로젝트를 압축해서 보낼 때, 아래 폴더와 파일만 보내면 됩니다.

src/
include/
Makefile, 
.vscode/, 
bin/ 폴더 등은 
macOS 환경 전용이므로 Windows 사용자에게는 필요하지 않습니다.

```
